Pourquoi le système précédent (upload + localStorage) a été retiré :
Les images uploadées via ce système n'étaient enregistrées que dans TON navigateur. Un visiteur externe — un recruteur — ouvrant cette page depuis son propre ordinateur voyait toujours "Aucune image", quoi que tu aies uploadé de ton côté. Ce n'était pas un bug mineur, la page ne pouvait structurellement pas remplir son rôle.

Comment remplir chaque emplacement, méthode qui fonctionne réellement :
1. Dans le repo GitHub, crée un dossier certs/ à la racine (à côté de index.html) s'il n'existe pas déjà.
2. Dépose chaque certificat en image (JPG/PNG) avec le nom exact indiqué sous chaque emplacement ci-dessus.
3. GitHub Pages sert le fichier automatiquement — aucune modification de code nécessaire, aucun risque de casser un lien.
⚠ Tant qu'un fichier n'est pas déposé, l'encadré en pointillé reste affiché tel quel — c'est voulu, ça ne casse rien visuellement, contrairement à un lien mort ou une iframe vide.
